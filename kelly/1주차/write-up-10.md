## ❓질문

```text
왜 HTTP는 TCP를 사용하나요?
```

---
## 😎 답변 및 꼬리 질문

### 답변

HTTP가 전통적으로 TCP를 채택한 근본적인 이유는 **웹 데이터의 특성상 '단 1비트의 데이터 손실이나 순서 뒤바뀜도 허용되지 않는 절대적인 신뢰성'이 필요하기 때문**입니다.

웹을 구성하는 HTML, CSS, JavaScript 코드나 JSON 데이터, 이미지 등은 전송 과정에서 일부 패킷이 유실되거나 순서가 어긋나면 화면이 완전히 깨지거나 프로그램이 비정상 종료되는 치명적인 문제가 발생합니다.

따라서 HTTP는 전송 계층에서 자체적으로 데이터의 신뢰성을 완벽히 보장해 주는 TCP를 선택했습니다. TCP는 **3-Way Handshake**를 통해 양단간의 연결을 확실히 수립한 후 데이터를 전송하며, **시퀀스 번호와 ACK 번호**를 이용해 패킷의 누락을 감지하고 재전송하는 **오류 제어**를 수행합니다. 또한, 수신 측의 처리 속도에 맞춰 데이터 양을 조절하는 **흐름 제어**, 네트워크의 혼잡 상태에 따라 전송 속도를 줄이는 **혼잡 제어** 메커니즘을 내장하고 있습니다.

즉, 초기 HTTP 개발자들은 신뢰성, 흐름 제어, 혼잡 제어 같은 복잡한 네트워크 문제를 하부 계층인 TCP에 전부 위임함으로써, 애플리케이션 계층인 HTTP 프로토콜 자체를 훨씬 단순하고 확장성 있게 설계할 수 있었습니다.

### 꼬리 질문

- 🤗 스포 방지!

  HTTP와 TCP의 끈끈한 관계를 명확히 설명했기 때문에, 면접관은 "그렇다면 왜 이제 와서 HTTP/3는 TCP를 버렸는가?"로 연결하거나, TCP의 고질적인 성능 제약에 대해 깊이 파고들 것입니다.

    - [ ]  방금 TCP가 흐름 제어와 혼잡 제어를 해줘서 HTTP가 안전하게 데이터를 보낼 수 있다고 하셨는데, 대규모 트래픽이 몰리는 상황에서 TCP의 '느린 시작(Slow Start)'이라는 혼잡 제어 알고리즘이 HTTP 통신 성능에 어떤 초기 병목을 유발할 수 있을까요?
    - [ ]  TCP의 신뢰성 보장 메커니즘이 웹에는 필수적이지만, 동영상 스트리밍 서비스(예: 유튜브, 넷플릭스)나 실시간 온라인 게임 같은 분야에서는 왜 여전히 TCP보다 UDP 기반의 커스텀 프로토콜을 선호하는 경향이 있을까요?
    - [ ]  결국 HTTP/3는 TCP의 신뢰성을 포기한 것이 아니라, 그 신뢰성 보장 역할을 하부 계층(TCP)이 아닌 **애플리케이션 계층(QUIC 프로토콜)으로 끌어올려 직접 구현**한 것입니다. 이렇게 신뢰성 구현 위치를 위로 올렸을 때 얻을 수 있는 아키텍처적 이점은 무엇일까요? (예: OS 커널 업데이트 없이 프로토콜 고도화 가능 등)

## 🤔 질문 의도 및 답변 분석

### 질문 의도

이 질문은 최신 기술(HTTP/3, UDP)에 매몰되지 않고, **컴퓨터 네트워크의 가장 핵심 가치인 '신뢰성'과 웹(Web) 데이터의 특성을 올바르게 연결하여 설명할 수 있는가**를 검증하기 위한 것입니다.

면접관이 듣고 싶어 하는 골드 답변은 "웹 데이터(텍스트, 코드, 이미지)는 단 1비트의 손실도 치명적이라는 특성"을 짚고, 이를 완벽하게 보장해 주는 TCP의 3대 핵심 메커니즘(흐름 제어, 혼잡 제어, 오류 제어)을 엮어 웹 생태계가 TCP 위에서 안전하게 성장할 수 있었던 이유를 논리적으로 설명하는 것입니다.

### 답변 분석

- **웹 데이터의 '비즈니스적 특성' 명시**: 단순히 "TCP가 신뢰성이 좋아서요"라고 답하면 실버 답변입니다. 골드 답변은 "HTML, 코드, JSON 데이터는 1비트만 깨져도 치명적이다"라는 웹 리소스의 특성을 명확히 짚어 왜 신뢰성이 최우선 과제였는지 당위성을 부여했습니다.
- **TCP 핵심 메커니즘의 정교한 나열**: TCP가 신뢰성을 보장하기 위해 내부적으로 수행하는 3대 기능(흐름 제어, 혼잡 제어, 오류 제어)과 구체적인 수단(ACK, 시퀀스 번호)을 정확하게 배치하여 네트워크 기본기가 완벽함을 보여줍니다.
- **계층 구조(OSI 7 Layer) 관점의 아키텍처적 해석**: 하부 계층(TCP)에 무거운 책임을 위임함으로써 상위 계층(HTTP)이 경량화될 수 있었다는 엔지니어링 관점의 해석을 덧붙여 답변의 깊이를 더했습니다.

### 핵심 키워드

- 웹 데이터의 무결성 (1비트의 손실도 치명적)
- TCP의 신뢰성 보장 (흐름 제어, 혼잡 제어, 오류 제어)
- 상위 계층(HTTP)의 역할 분리와 단순화
