## 😎 답변 및 꼬리 질문

### 답변

**프로세스 주소공간**은 운영체제 커널이 프로세스를 실행할 때 할당하는 독립된 '가상 메모리 주소 공간'입니다.

만약 프로세스가 실제 물리 RAM의 주소를 직접 찌르게 설계된다면, 해킹 프로그램이 옆집 프로세스의 금융 데이터 메모리를 훔쳐보거나 OS 영역을 파괴하는 대참사가 날 수 있습니다. 따라서 리눅스 커널은 가상 메모리 시스템을 도입하여, 모든 프로세스가 **"나는 0번지부터 시작하는 수 GB짜리 청정한 독립 메모리를 혼자 독점하고 있다"고 착각하게 만드는 거대한 격리 방어벽**을 세워줍니다.

이 가상 주소 공간은 효율적인 자원 관리를 위해 크게 5가지 세그먼트 영역으로 분할됩니다.

### 1. 정적 영역 (컴파일 타임에 크기가 고정되는 영역)

- **Code (Text) 영역**: 개발자가 작성한 소스 코드가 기계어 바이너리 형태로 저장되는 공간입니다. 런타임에 코드가 오염되거나 바뀌면 안 되기 때문에, 오직 읽기 전용(**Read-Only**) 권한만 부여됩니다.
- **Data 영역**: `int a = 10;` 처럼 **초기값이 있는 전역 변수나 정적(static) 변수**가 저장됩니다. 실행 파일 내부에 변수 값이 고정되어 포함되므로 프로그램 용량에 영향을 줍니다.
- **BSS 영역**: `int b;` 처럼 **초기화하지 않은 전역/정적 변수**가 모입니다. 디스크 용량을 아끼기 위해 실행 파일에는 크기 정보만 기록했다가, 런타임에 커널이 0으로 일괄 동적 초기화합니다.

### 2. 동적 영역 (런타임 실행 중에 크기가 실시간으로 변하는 영역)

- **Heap 영역**: 프로그래머가 필요에 의해 동적으로 메모리를 할당하는 공간입니다. (자바의 `new Object()`, C언어의 `malloc()`). 주소값이 **낮은 주소에서 높은 주소 방향으로 쌓이며**, 가비지 컬렉터(GC)나 개발자가 직접 수거할 때까지 메모리가 유지됩니다.
- **Stack 영역**: 함수가 호출될 때 사용되는 로컬 지역 변수, 매개 변수, 리턴 주소 등이 **스택 프레임(Stack Frame)** 단위로 저장됩니다. 주소값이 **높은 주소에서 낮은 주소 방향으로 내려오며**, 함수가 종료되는 순간 즉시 자동으로 자원이 파괴(POP)됩니다.

> **※ 아키텍처적 충돌 방지 설계**
힙은 위로 자라고, 스택은 아래로 자라도록 정반대 방향으로 설계되었습니다. 이는 런타임에 메모리를 가변적으로 최대한 활용하되, 서로 영역을 침범하여 발생하는 에러인 **Heap Overflow**와 **Stack Overflow**를 방지하기 위한 영리한 대칭 구조 설계입니다.
>

결론적으로 프로세스 주소공간은 가상 메모리 아키텍처 위에서 **정적 자원(Code/Data/BSS)과 동적 자원(Heap/Stack)을 효율적으로 분리하여 무결성을 유지하고, 프로세스 간 자원 침범을 원천 차단하는 OS의 핵심 격리 장치**입니다.

## 🤔 질문 의도 및 답변 분석

### 질문 의도

면접관이 확인하고 싶은 골드 포인트는 두 가지입니다.

1. **가상 주소 공간(Virtual Address Space)의 이해** : 프로세스가 왜 물리 메모리 주소를 직접 쓰지 않고 OS가 제공하는 0번지부터 시작하는 가상 주소를 쓰는지 그 당위성을 아는가.
2. **5대 영역(Segment)의 생명 주기와 특징 식별** : 정적 영역(Code, Data, BSS)과 동적 영역(Heap, Stack)의 메모리 증가 방향 및 컴파일/런타임 결정 시점의 차이를 완벽히 대조할 수 있는가.

### 답변 분석

- **가상 메모리 도입의 당위성 명시**: 단순히 "5대 영역이 있다"는 백과사전식 답변을 넘어, **왜 물리 RAM 주소를 숨기고 가상 주소 공간을 쪼개어 주어야 하는지** 보안과 격리라는 근본적 인과관계를 정확히 서술했습니다.
- **힙과 스택의 성장 방향 대조**: 힙은 위로, 스택은 아래로 자라는 대칭성 배치의 하드웨어적 설계 의도(오버플로우 방지 및 주소 공간 극대화)를 찔러 깊은 점수를 획득했습니다.

### 핵심 키워드

- 커널이 프로세스별로 할당하는 독립적이고 안전한 격리 공간인 가상 주소 공간
- 정적 영역(Code, Data, BSS)과 런타임 동적 변동 영역(Heap, Stack)의 라이프 사이클 차이
- 힙(낮은 주소 → 높은 주소)과 스택(높은 주소 → 낮은 주소)의 대칭적 성장 구조
