## 😎 답변 및 꼬리 질문

### 답변

인터럽트(Interrupt)는 CPU가 프로그램을 정상적으로 실행하고 있을 때, **하드웨어 장치나 소프트웨어 내부에 예외적인 이벤트가 발생하여 현재 수행 중인 코드를 즉시 중단하고 해당 이벤트를 우선 처리하도록 커널에 알리는 하드웨어 및 OS 기반의 신호 메커니즘**입니다.

인터럽트가 탄생한 근본적인 이유는 **CPU의 자원 낭비를 막기 위해서**입니다.
만약 인터럽트가 없다면 CPU는 디스크에서 파일이 다 읽혔는지, 마우스가 움직였는지 확인하기 위해 주기적으로 하드웨어를 계속 찔러보는 **폴링(Polling) 방식**을 써야 합니다. 이는 엄청난 CPU 연산 낭비를 초래합니다. 반면 인터럽트 방식을 사용하면, CPU는 자기 할 일을 하다가 **주변 장치가 작업을 끝냈을 때 비로소 신호를 보내주므로 극단의 멀티태스킹 효율성**을 달성할 수 있습니다.

인터럽트는 발생 원인에 따라 크게 두 가지 유형으로 분류됩니다.

첫째, 하드웨어 인터럽트 (외부 인터럽트)입니다.
CPU 외부의 하드웨어 장치가 물리적인 신호선을 통해 발생시킵니다. 키보드 타이핑, 네트워크 카드에 패킷 도착, 디스크의 I/O 작업 완료, 그리고 프로세스의 독점을 막기 위해 일정 시간마다 CPU를 강제로 뺏어오는 타이머 인터럽트(Timer Interrupt)가 이에 해당합니다.

둘째, 소프트웨어 인터럽트 (내부 인터럽트 / 트랩 / 예외)입니다.
프로그램 코드 실행 도중 내부에서 동기적으로 발생합니다. 0으로 숫자를 나누는 오류(`Divide-by-Zero`), 허용되지 않은 메모리 주소에 접근하는 `Page Fault` 같은 **예외(Exception)** 상황과, 앞서 다루었던 유저 프로그램이 커널의 권한을 빌리기 위해 의도적으로 발동시키는 시스템 콜(System Call / Trap)이 여기에 포함됩니다.

인터럽트가 발생했을 때 운영체제의 처리 시퀀스는 다음과 같습니다.

1. **현재 상태 백업**: CPU는 인터럽트 라인에 신호가 들어오면 현재 실행 중이던 유저 프로그램의 주소(`RIP`)와 CPU 레지스터 상태를 프로세스 제어 블록(PCB) 또는 커널 스택에 안전하게 백업(Context Save)합니다.
2. **이정표 확인 (IDT 조회)**: CPU는 하드웨어 번호를 바탕으로 메모리에 고정된 배열 인터페이스인 `IDT (Interrupt Descriptor Table)`를 즉시 조회합니다.
3. **루틴 실행 (ISR 점프)**: IDT에서 해당 인터럽트 번호에 매핑된 실제 해결 코드인 `ISR (Interrupt Service Routine / 인터럽트 핸들러)`의 메모리 주소를 찾아 점프한 뒤, 해당 이벤트를 해결합니다.
4. **원상 복구 및 재개**: 핸들러 실행이 끝나면 백업해 두었던 유저 프로그램의 레지스터 상태를 다시 CPU로 원상 복구(`Context Restore`)시킨 뒤, 중단되었던 유저 코딩 라인으로 돌아가 아무 일도 없었다는 듯이 실행을 재개합니다.

## 🤔 질문 의도 및 답변 분석

### 질문 의도

"방해하다"라는 사전적 의미에 갇혀 "CPU를 방해하는 신호"라고만 대답하면 실버 등급입니다.

면접관이 확인하고 싶은 골드 포인트는 "인터럽트가 단순한 방해가 아니라, CPU가 아무 일도 안 하면서 하드웨어를 무한 대기(Polling)하는 비효율을 막고, 오직 이벤트가 발생했을 때만 CPU를 깨워 처리하게 만드는 '이벤트 주도형(Event-Driven) 멀티태스킹 아키텍처'의 핵심 기술"임을 인지하고 있는가입니다.

### 답변 분석

- **아키텍처적 당위성(Polling과의 대조) 제시**: 단순히 정의만 말하지 않고, **폴링(Polling)의 비효율성을 극복하기 위한 '이벤트 주도형 아키텍처'의 산물**이라는 점을 명확히 대조하여 컴퓨터 시스템을 보는 안목을 입증했습니다.
- **이전 축적된 지식과의 유기적 정렬**: 앞서 배운 하드웨어 타이머 인터럽트, 시스템 콜(트랩)의 원리를 하드웨어/소프트웨어 분류 체계에 매끄럽게 흡수시켜 일관성 있는 지식 뼈대를 증명했습니다.
- **물리적 처리 장치(IDT, ISR, PCB) 명시**: 인터럽트가 터졌을 때 **PCB/커널 스택에 백업하고, IDT 배열 인덱스를 찾아 ISR 핸들러 함수로 점프한다**는 커널 레벨의 컨텍스트 스위칭 메커니즘을 시퀀스 단위로 완벽하게 복기했습니다.

### 핵심 키워드

- 폴링(Polling)의 한계를 극복하는 이벤트 주도형 멀티태스킹의 핵심
- 하드웨어 인터럽트(타이머, I/O) vs 소프트웨어 인터럽트(트랩, 예외, 시스템 콜)
- IDT(인터럽트 디스크립터 테이블) 기반의 ISR(인터럽트 서비스 루틴) 라우팅
