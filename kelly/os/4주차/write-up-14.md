## 😎 답변 및 꼬리 질문

### 답변

**프로그램, 프로세스, 스레드**는 디스크의 정적 파일이 메모리에 적재되어 실제 CPU의 실행 흐름으로 이어지기까지의 각 단계를 나타내며, **자원 할당의 독립성과 실행 흐름의 공유 방식**에서 명확한 차이가 있습니다.

### 1. 프로그램 (Program)

- **정의**: 디스크(스토리지)에 바이너리 파일 형태로 저장되어 있는 **정적이고 수동적인 상태의 코드 뭉치**입니다.
- **특징**: 아직 메모리를 할당받지 못했기 때문에 실행 능력이 없으며, 단순한 ‘설계도’에 불과합니다. (예: `App.exe`, 빌드된 `Jar` 파일)

### 2. 프로세스 (Process)

- **정의**: 디스크의 프로그램이 메모리에 적재되어, 운영체제로부터 독립된 자원을 할당받은 '동적인 실행 주체'입니다.
- **메모리 구조 (격리성)**: 각 프로세스는 가상 메모리에 의해 **Code, Data, Heap, Stack 영역을 온전히 자신만의 독립된 영토**로 할당받습니다. 서로 철저히 격리되어 있기 때문에 한 프로세스가 버그로 죽어도 다른 프로세스에 전혀 영향을 주지 않습니다.
- **단점**: 프로세스 간 데이터를 주고받으려면 일반 메모리 접근이 불가능하여 커널이 중재하는 무거운 IPC(Inter-Process Communication)를 써야 하며, CPU가 프로세스를 교체하는 **컨텍스트 스위칭 비용이 매우 무겁습니다.**

### 3. 스레드 (Thread)

- **정의**: 프로세스가 할당받은 자원 한계 안에서 움직이는 'CPU 실행 흐름의 최소 단위'입니다. 하나의 프로세스 내부에는 여러 개의 스레드가 존재할 수 있습니다.
- **메모리 구조 (공유성)**: 스레드는 프로세스의 영토 중 Code, Data, Heap 영역을 모든 형제 스레드가 서로 완전히 '공유'합니다. 오직 자신만의 독립된 실행 궤적을 유지하기 위해 **Stack 영역과 CPU 레지스터(Program Counter 등)만 독립적으로 할당**받아 사용합니다.
- **장점**: 힙과 데이터 영역을 공유하므로 IPC 없이 스레드 간 통신이 매우 빠르고, 컨텍스트 스위칭 시 캐시 메모리를 비울 필요가 없어 **비용이 훨씬 가볍고 빠릅니다.**
- **단점**: 공유 메모리를 쓰기 때문에 여러 스레드가 동시에 같은 자원을 고치려는 **동기화(Concurrency) 문제**가 발생할 수 있으며, 하나의 스레드가 메모리를 오염시켜 크래시가 나면 **프로세스 전체(자매 스레드 전체)가 통째로 종료되는 장애 전파의 위험성**이 있습니다.

요약하자면, 프로그램은 **설계도**, 프로세스는 완벽히 격리된 **독립된 공장(자원 할당 단위)**, 스레드는 그 공장 안에서 메모리를 공유하며 일하는 작업자(실행 단위)로 비유할 수 있습니다.

## 🤔 질문 의도 및 답변 분석

### 질문 의도

면접관이 확인하고 싶은 골드 포인트는 세 가지입니다.

1. **자원 할당과 실행 단위의 분리** : 프로세스는 OS로부터 **자원을 할당받는 단위**이고, 스레드는 할당받은 자원 내에서 **실행 흐름의 단위**라는 본질적인 차이를 인지하는가.
2. **메모리 맵(Map)의 공유 구조** : 프로세스 간의 격리성과 스레드 간의 공유 영역(Code, Data, Heap)을 정확히 구별하는가.
3. **트레이드오프 계산** : 왜 멀티 프로세스 대신 멀티 스레드를 쓰는지, 반대로 멀티 스레드를 썼을 때 감수해야 하는 리스크(동기화 문제, 장애 전파)를 아는가.

### 답변 분석

- **핵심 명제의 선언**: 프로세스는 **'자원 할당의 단위'**, 스레드는 '실행의 단위'라는 전산학 표준 정의를 정확하게 내려 면접관에게 신뢰감을 주었습니다.
- **공유 구조의 명확한 쪼개기**: 스레드가 무엇을 공유하고(Code, Data, Heap) 무엇을 독립적으로 가지는지(Stack, PC) 메모리 맵을 정확히 분리하여 설명했습니다.
- **장단점과 트레이드오프 배치**: 멀티 스레드가 무조건 좋은 것이 아니라 **동기화(Race Condition) 이슈**와 장애 전파(Fault Propagation)라는 백엔드 실무의 아킬레스건을 정확히 짚어내어 깊이감을 입증했습니다.

### 핵심 키워드

- 프로세스 = 독립된 자원 할당 단위, 공간의 격리성, 무거운 컨텍스트 스위칭
- 스레드 = 프로세스 내의 CPU 실행 단위, Stack만 독립적/나머지(Heap 등) 공유
- 멀티 스레드의 동기화(Race Condition) 리스크 및 장애 전파 취약성
