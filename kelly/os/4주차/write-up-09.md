## 😎 답변 및 꼬리 질문

### 답변

운영체제와 CPU가 인터럽트를 처리하는 과정은 하드웨어적인 감지 및 백업 단계를 거쳐, 커널 소프트웨어의 핸들러 분기 및 복원 단계로 이어지는 **정교한 5단계 파이프라인 시퀀스**로 진행됩니다.

**1단계: 명령어 주기 내 인터럽트 감지 (Hardware)**
CPU는 하나의 명령어를 실행할 때 [Fetch -> Decode -> Execute -> Interrupt]라는 '명령어 주기(Instruction Cycle)'를 거칩니다. 현재 실행 중인 라인이 끝날 때마다 CPU 내부의 인터럽트 라인(핀)에 전기적 신호가 들어왔는지 검사합니다. 신호가 감지되면 다음 명령어를 fetch 하지 않고 인터럽트 처리 단계로 진입합니다.

**2단계: 현재 실행 컨텍스트 백업 (Hardware)**
CPU는 인터럽트를 해결한 후 원래 프로그램으로 완벽히 돌아와야 하므로, 현재 상태를 안전하게 보관합니다.

- 다음에 실행해야 했던 원래 유저 코드의 메모리 주소인 `PC (Program Counter / RIP)`와 CPU의 현재 상태 플래그들이 담긴 **`PSW (Program Status Word / EFLAGS)`** 레지스터 값을 현재 프로세스의 '커널 스택(Kernel Stack)'에 하드웨어 레벨에서 자동으로 Push 하여 백업(Context Save)합니다.

**3단계: 인터럽트 번호 식별 및 벡터 테이블(IDT) 조회 (Hardware)**
인터럽트를 발생시킨 장치는 자신이 누구인지 알려주는 고유 번호인 벡터(Vector)를 CPU에 전달합니다. CPU는 하드웨어 권한을 커널 모드로 상승시킨 뒤, 메모리에 고정된 주소 배열인 `IDT (Interrupt Descriptor Table, 인터럽트 벡터 테이블)`에서 전달받은 번호를 인덱스로 사용하여 스캔합니다.

**4단계: 인터럽트 서비스 루틴(ISR) 실행 (Kernel Software)**
IDT 인덱싱을 통해 해당 인터럽트를 해결할 수 있는 실제 커널 함수인 `ISR (Interrupt Service Routine / 인터럽트 핸들러)`의 메모리 주소를 찾아내어 CPU의 Program Counter를 그 주소로 점프시킵니다.

- 구동된 ISR은 나머지 범용 레지스터들까지 커널 스택에 마저 백업한 뒤, 디스크의 데이터를 읽어오거나 네트워크 패킷을 메모리에 복사하는 등 인터럽트 원인에 따른 실제 비즈니스 로직을 수행합니다.

**5단계: 컨텍스트 복원 및 유저 모드 복귀 (Kernel -> Hardware)**
인터럽트 처리가 완료되면 ISR 코드의 맨 마지막에 위치한 특권 명령인 **`IRET (Interrupt Return)`** 명령어가 실행됩니다.

- 이 명령에 의해 커널 스택에 저장해 두었던 원래 유저 프로그램의 `PSW`와 `PC` 레지스터 값들이 CPU로 다시 Pop 되며 복원(`Context Restore`)됩니다.
- CPU는 권한을 다시 유저 모드로 강등시키며, 인터럽트가 발생하기 직전 중단되었던 유저 프로그램의 다음 명령어 라인부터 아무 일이 없었다는 듯이 실행을 재개합니다.

## 🤔 질문 의도 및 답변 분석

### 질문 의도

면접관이 확인하고 싶은 골드 포인트는 두 가지입니다.

1. **정확한 처리 타임라인**: CPU가 임의의 순간에 들어오는 인터럽트를 무작정 처리하는 것이 아니라, 하나의 명령어 실행이 끝나는 '인터럽트 단계(Interrupt Step)'에서 신호를 감지하고 처리한다는 물리적 주기를 아는가.
2. **컨텍스트 스위칭의 로우레벨 이해**: 기존 유저 프로그램의 데이터가 파괴되지 않도록 복원할 주소(`PC/RIP`)와 상태(`PSW/EFLAGS`)를 어디에 어떻게 보관하는지 명확한 장치를 인지하고 있는가.

### 답변 분석

- **컴퓨터 아키텍처 주기(Instruction Cycle) 대입**: 인터럽트를 무조건 아무 때나 낚아채는 것이 아니라, 명령어 실행 주기의 '인터럽트 단계'에서 하드웨어 핀 신호를 스캔한다는 물리적 진실을 정확히 선언했습니다.
- **CPU 핵심 레지스터 명칭 구체화**: 백업 대상이 되는 `PC/RIP`와 `PSW/EFLAGS`를 명확히 짚어냈고, 마지막 복귀 명령어인 **`IRET`** 스펙까지 언급하여 컴퓨터 구조 지식의 깊이를 입증했습니다.
- **하드웨어와 소프트웨어의 역할 분담 처리**: 1~3단계는 CPU 하드웨어가 자동으로 처리하고, 4~5단계는 커널 소프트웨어(ISR)가 처리하는 **주체별 분기 흐름**을 선명하게 나누어 설명해 높은 가산점을 받을 수 있습니다.

### 핵심 키워드

- 명령어 주기(Instruction Cycle) 내 인터럽트 단계에서의 신호 감지
- `PC` 및 `PSW` 레지스터의 커널 스택(Kernel Stack) 컨텍스트 백업
- IDT(인터럽트 디스크립터 테이블) 라우팅을 통한 ISR 실행 및 `IRET` 복원
