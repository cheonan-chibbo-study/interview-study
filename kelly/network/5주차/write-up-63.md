## ❓질문

```text
수많은 사람들이 유동 IP를 사용하고 있지만, 수많은 공유기에서는 고정 주소를 제공하는 기능이 이미 존재합니다. 어떻게 가능한 걸까요?
```

## 😎 답변 및 꼬리 질문

### 답변

"수많은 사용자가 통신사로부터 유동 IP를 할당받으면서도, 가정이나 사무실의 공유기 내부에서 특정 컴퓨터에 고정 주소를 줄 수 있는 이유는 **공유기가 외부 인터넷망(WAN)과 내부 사설망(LAN)을 철저히 분리하여 다스리는 독립된 라우터이자 DHCP 서버이기 때문**입니다.

이 마법 같은 아키텍처가 구동되는 원리는 크게 2가지 레이어로 나누어 설명할 수 있습니다.

#### 1. 외부 유동 IP와 내부 사설 IP의 분리 (NAT)

- **외부 망 (WAN)**: SKT, KT, LGU+ 같은 통신사(ISP)는 주소 고갈을 막기 위해 공유기의 외부 포트(`WAN`)에 주기적으로 변하는 **공인 유동 IP**를 대여(DHCP)해 줍니다.
- **내부 망 (LAN)**: 하지만 공유기 안쪽 세상은 완전히 독립된 영토입니다. 공유기는 NAT(네트워크 주소 변환) 기술을 사용해 외부 공인 IP가 유동적으로 바뀌든 말든 상관없이, 내부 기기들에게는 `192.168.0.2`, `192.168.0.3` 같은 **고유한 사설 IP 대역을 독자적으로 선포하고 지배**합니다.

#### 2. 공유기 내부 DHCP 서버의 'MAC 주소 고정 매핑' 기능 (Static Lease)

공유기 내부에는 기기들이 와이파이를 잡을 때마다 사설 IP를 자동으로 나눠주는 **DHCP 서버 엔진**이 탑재되어 있습니다. 일반적으로 이 엔진도 돌려막기식 유동 할당을 하지만, 공유기 설정에서 '고정 주소 제공 기능'을 켜면 다음과 같은 커널 주소록 바인딩이 일어납니다.

- **하드웨어 식별**: 전 세계 모든 네트워크 기기(노트북, 스마트폰 등)의 랜카드에는 절대 변하지 않는 고유한 주민등록번호인 MAC 주소(물리 주소)가 박혀 있습니다.
- **정적 바인딩 (Static Binding)**: 사용자가 공유기 관리자 페이지에서 "내 맥북의 MAC 주소(`AA:BB:CC:...`)에게는 무조건 사설 IP `192.168.0.100`번을 고정으로 배포해라"고 규칙을 선언해 둡니다.
- **런타임 동작**: 이후 내 맥북이 켜지면서 공유기에게 "나IP 주소 하나만 빌려줘"라고 DHCP 요청을 던지면, 공유기는 맥북의 MAC 주소를 확인하고 주소록 테이블을 매핑하여 **대여 시간(Lease Time) 제한이 없는 영구적인 고정 사설 IP를 디멀티플렉싱하여 딱 집어 배달**해 줍니다.

결론적으로, 통신사가 공유기에게 주는 외부 IP가 유동적으로 변하는 것과 관계없이, 공유기는 내부 사설망이라는 자신만의 영토 안에서 **기기의 하드웨어 고유 식별자인 MAC 주소와 특정 사설 IP를 1:1로 정적으로 묶어버리는 DHCP 고정 할당 기술을 사용하기 때문에 가능한 현상**입니다."

## 🤔 질문 의도 및 답변 분석

### 질문 의도

면접관이 확인하고 싶은 골드 포인트는 두 가지입니다.

1. **공인 IP(WAN)와 사설 IP(LAN)의 경계선 인지**: 통신사가 주는 유동 IP는 공유기의 외부 단자(`WAN`)에 꽂히는 공인 IP이고, 공유기 하부 기기들이 나누어 갖는 IP는 사설 IP(`LAN`)임을 명확히 구분하는가.
2. **DHCP 고정 매핑(Static Binding) 메커니즘**: 공유기 내부의 DHCP 서버가 기기의 고유 하드웨어 주소인 **MAC 주소**를 나침반 삼아 특정 사설 IP를 영구적으로 묶어버리는 원리를 아는가.

### 답변 분석

- **네트워크 바운더리의 선명한 경계선 정돈**: 외부 공인 IP와 내부 사설 IP의 통제권 주체(통신사 vs 공유기)가 완전히 별개라는 공간적 분리를 선언하여 논리적 의문점을 완벽하게 해소했습니다.
- **물리 주소(MAC)와 논리 주소(IP)의 바인딩 원리 입증**: 공유기가 고정 주소를 주는 진짜 하부 기술 명세인 DHCP 정적 매핑(Static Binding / IP-MAC Reservation)의 인과관계를 정확하게 논증했습니다.

### 핵심 키워드

- 외부 통신사망(WAN 공인 유동 IP)과 내부 공유기망(LAN 사설 IP)의 NAT 격리 구조
- 기기의 변하지 않는 고유 하드웨어 식별자인 MAC 주소(물리 주소) 활용
- 공유기 내장 DHCP 서버의 IP-MAC 정적 바인딩(Static Lease) 매커니즘
