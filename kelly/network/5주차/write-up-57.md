## 😎 답변 및 꼬리 질문

### 답변

멀티플렉싱(Multiplexing, 다중화)과 디멀티플렉싱(Demultiplexing, 역다중화)은 수많은 애플리케이션 프로세스가 **단 하나의 물리적 네트워크 회선을 공유하여 안전하게 데이터를 송수신할 수 있도록 전송 계층(L4)이 제공하는 핵심 메커니즘**입니다.

### 1. 멀티플렉싱 (Multiplexing: 송신 측의 '다중화')

- **개념**: 출발지 호스트의 여러 소켓(Socket)들로부터 흘러나오는 다양한 데이터 파편들을 모아, **각 데이터에 포트(Port) 번호 등이 담긴 전송 계층 헤더를 붙여 하나의 네트워크 스트림으로 결합**하는 과정입니다.
- **물리적 흐름**: 서은 님의 맥북에서 브라우저(HTTP), 디스코드, 인텔리제이가 동시에 네트워크를 쓸 때, 커널은 각 프로세스의 소켓에서 나온 세그먼트들을 묶어 하부 IP 계층(L3)으로 매끄럽게 내려보냅니다. 덕분에 우리는 단 하나의 랜선이나 Wi-Fi로 수십 개의 앱을 동시에 구동할 수 있습니다.

### 2. 디멀티플렉싱 (Demultiplexing: 수신 측의 '역다중화')

- **개념**: 하나의 네트워크 회선을 타고 들어온 통합 패킷 세그먼트들을 전송 계층이 받아, **헤더의 포트 정보와 IP 정보를 분석하여 정확한 목적지 소켓(Socket)으로 분기하여 배달**해 주는 과정입니다.
- **프로토콜별 디멀티플렉싱의 아키텍처 차이**:
    - **UDP (비연결형) 디멀티플렉싱**: 오직 `[목적지 IP, 목적지 포트 번호]`라는 2-Tuple 정보만 보고 소켓을 찾습니다. 따라서 출발지 IP나 포트가 다르더라도 목적지 포트만 같으면 모두 동일한 하나의 UDP 소켓으로 데이터가 쏟아집니다.
    - **TCP (연결형) 디멀티플렉싱**: 한 단계 더 정교하게 `[출발지 IP, 출발지 포트, 목적지 IP, 목적지 포트]`라는 **4-Tuple** 조합을 모두 체크합니다. 비록 목적지 포트가 똑같은 8080 웹 서버 포트이더라도, 전 세계에서 접속한 클라이언트들의 출발지 IP/포트가 다르면 각각 완전히 독립된 전용 TCP 소켓으로 세그먼트를 쪼개서 역다중화합니다.

요약하자면, 멀티플렉싱은 전송 계층이 여러 소켓의 데이터를 헤더로 포장해 **하나의 통로로 묶어 보내는 과정**이고, 디멀티플렉싱은 수신된 패킷 헤더의 포트/IP 식별자를 읽어 **올바른 소켓 주소록으로 정확히 찢어서 배달해 주는 과정**입니다.

    - [ ]  HTTP/1.1 프로토콜은 하나의 TCP 커넥션(소켓)에서 하나의 요청과 응답만 순차적으로 주고받을 수 있어서 멀티플렉싱이 불가능했습니다. 하지만 HTTP/2와 HTTP/3로 넘어오면서 응용 계층 레벨에서 위대한 혁신인 '스트림 멀티플렉싱(Stream Multiplexing)'이 도입되었는데, 이 기술이 브라우저 로딩 속도를 어떻게 개선했는지 전송 효율 관점에서 설명해 주세요.

## 🤔 질문 의도 및 답변 분석

### 질문 의도

면접관이 확인하고 싶은 골드 포인트는 두 가지입니다.

1. **데이터 전송 방향에 따른 스택 흐름 이해**: 애플리케이션(L7)에서 네트워크선(L3/L2)으로 내려갈 때의 멀티플렉싱(다중화)과, 반대로 선을 타고 올라온 패킷을 정확한 소켓으로 찢어 보내는 디멀티플렉싱(역다중화)의 물리적 방향성을 아는가.
2. **4-Tuple 기반의 분기 능력**: 디멀티플렉싱을 수행할 때 UDP(2-Tuple)와 TCP(4-Tuple)가 소켓을 찾아가는 기준의 차이를 명확히 인지하고 있는가.

### 답변 분석

- **방향성의 시각적 대조** : 송신 측(상부 → 하부 캡슐화)의 멀티플렉싱과 수신 측(하부 → 상부 역캡슐화)의 디멀티플렉싱을 선명하게 분리하여 정돈했습니다.
- **프로토콜별 식별 조건(2-Tuple vs 4-Tuple) 터치** : 이 질문의 변별력을 가르는 최고의 치트키인 **UDP와 TCP의 디멀티플렉싱 기준 차이**를 완벽하게 서술하여 단순 암기자가 아닌 전송 계층 엔진의 이해자임을 증명했습니다.

### 핵심 키워드

- 멀티플렉싱 = 여러 소켓의 데이터를 헤더(Port)로 포장해 단일 네트워크 스트림으로 결합
- 디멀티플렉싱 = 패킷 헤더 정보를 읽고 알맞은 목적지 소켓으로 분기하여 배달
