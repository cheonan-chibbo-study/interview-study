## 😎 답변 및 꼬리 질문

### 답변

**IP(Internet Protocol) 주소**는 인터넷망에 연결된 모든 컴퓨터와 네트워크 장비들이 서로를 식별하고 데이터를 정확하게 주고받을 수 있도록, 운영체제와 네트워크 계층(L3) 인터페이스가 부여한 '논리적인 가상 주소'입니다.

쉽게 비유하자면, 랜카드에 새겨진 주민등록번호 같은 물리 주소인 **MAC 주소**와 달리, IP 주소는 컴퓨터가 전 세계 어느 인터넷망에 꽂혀 있느냐에 따라 유동적으로 변하는 '우편물 배송 주소(예: 대한민국 천안시 ...)'라고 볼 수 있습니다.

IP 주소의 핵심 기능과 작동 메커니즘은 크게 두 가지로 요약할 수 있습니다.

### 1. 주소 지정을 위한 계층적 식별 기능 (Addressing)

IP 주소(IPv4 기준 32비트)는 단순히 무작위 숫자의 나열이 아니라, 서브넷 마스크(Subnet Mask)를 이정표 삼아 `[네트워크 주소 (Network ID)]`와 `[호스트 주소 (Host ID)]`라는 철저한 계층 구조로 나뉩니다.

- **네트워크 주소**: 해당 컴퓨터가 전 세계의 수많은 네트워크 망 중 '어떤 동네(라우터 하부 망)'에 소속되어 있는지를 알려줍니다.
- **호스트 주소**: 그 특정 동네 망 안에서 '어떤 구체적인 기기(컴퓨터)'가 내 기기인지를 구별해 줍니다.
- 이 계층 구조 덕분에 전 세계의 라우터들은 수억 대의 IP를 일일이 외울 필요 없이, 앞부분의 '네트워크 주소'만 보고 패킷을 굵직굵직하게 던져주는 효율적인 주소 지정이 가능해집니다.

### 2. 목적지를 찾아가는 최적의 경로 제어 기능 (Routing)

- 라우팅(Routing)은 송신 측에서 출발한 데이터 패킷이 목적지 IP 주소에 도달할 수 있도록, 중간 경로에 있는 네비게이션 장치인 '라우터(Router)'들이 패킷의 목적지 IP를 검사하여 다음 최적의 경로(Next Hop)로 토스해 주는 기능입니다.
- 패킷이 거대한 인터넷 바다를 건너 서은 님의 스프링 부트 서버 포트 앞까지 무사히 유입될 수 있는 것은, 전송 레벨의 모든 이정표 가이드라인이 이 IP 주소의 라우팅 기능을 기반으로 설계되어 있기 때문입니다.

요약하자면, IP 주소는 인터넷 생태계에서 장치의 위치를 나타내는 **논리적 주소 식별자**이며, 이를 통해 패킷이 길을 잃지 않고 전 세계 네트워크망을 종단하며 목적지 프로세스 문앞까지 이동할 수 있도록 **주소 지정(Addressing)과 경로 제어(Routing)의 핵심 기능**을 수행합니다.

## 🤔 질문 의도 및 답변 분석

### 질문 의도

면접관이 확인하고 싶은 골드 포인트는 두 가지입니다.

1. **논리적 식별자와 위치성의 이해**: IP 주소는 고정된 번호가 아니라 컴퓨터가 속한 **네트워크의 위치**에 따라 유동적으로 변하는 '논리적 주소'임을 인지하고 있는가.
2. **계층적 라우팅 구조의 이해**: IP 주소가 전 세계 수억 대의 기기 중 내 컴퓨터를 찾기 위해 `[네트워크 ID + 호스트 ID]`라는 이정표 구조를 취하고 있음을 아는가.

### 답변 분석

- **물리 주소(MAC)와의 선명한 대조 비유**: 주민등록번호(MAC)와 우편 주소(IP)라는 완벽한 개념 비유를 도입하여 면접관의 직관적인 이해를 도왔습니다.
- **비트 단위 하부 구조 분해**: 서브넷 마스크를 매개로 `Network ID`와 `Host ID`가 어떻게 계층을 이루어 전 세계 라우팅 테이블의 폭발을 막아내는지 인프라적 당위성을 정확히 논증했습니다.

### 핵심 키워드

- 인터넷망에 연결된 기기들을 식별하기 위한 L3 논리적 주소 체계
- `Network ID`(소속 망)와 `Host ID`(망 내 기기) 구조를 통한 효율적 계층화
- 라우터 장비들이 최적의 이정표를 찾아 패킷을 배달하는 라우팅(Routing) 기능
