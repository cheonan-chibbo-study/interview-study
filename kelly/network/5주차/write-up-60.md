## ❓질문

```text
IPv6는 IPv4의 주소 고갈 문제를 해결하기 위해 만들어졌지만, 아직도 수많은 기기가 IPv4를 사용하고 있습니다. 고갈 문제를 어떻게 해결할 수 있을까요?
```

---
## 😎 답변 및 꼬리 질문

### 답변

IPv6가 완벽한 대안임에도 불구하고 여전히 IPv4가 널리 쓰이는 이유는, 기존 전 세계 네트워크 인프라와 레거시 장비들을 교체하는 비용과 호환성 장벽이 매우 크기 때문입니다. 따라서 현대 네트워크 엔지니어링에서는 **IPv4의 수명을 극단적으로 늘리기 위해 크게 3가지 기술적 방어선으로 주소 고갈 문제를 해결**하고 있습니다.

### 1. 사설 IP 체계와 NAT/PAT 기술의 도입 (가장 결정적인 해결책)

모든 컴퓨터가 전 세계에서 유일한 '공인 IP'를 가질 필요가 없도록, 폐쇄된 내부 망 안에서만 자유롭게 재사용할 수 있는 **사설 IP(Private IP)** 대역을 정의했습니다.

- **NAT (Network Address Translation)**: 공유기나 방화벽 등의 장비가 내부의 수많은 사설 IP들을 단 몇 개의 공인 IP로 변환하여 외부 인터넷과 통신하게 만듭니다.
- **PAT (Port Address Translation)**: NAT를 한 단계 더 고도화한 기술로, **단 1개의 공인 IP 뒤에 수천 개의 '포트 번호(Port)'를 일대일 매핑**합니다. 예를 들어 내부 사설 IP를 쓰는 컴퓨터 100대가 외부 웹 서버로 패킷을 보낼 때, 공인 IP는 똑같이 `1.1.1.1`을 쓰되 포트 번호를 `50001`, `50002` 등으로 각각 다르게 찢어서 내보냅니다. 이 덕분에 공인 IP 1개만으로도 수천 대의 기기가 동시에 인터넷을 쓸 수 있게 되어 주소 고갈을 획기적으로 막아냈습니다.

### 2. 클래스리스(Classless) 기반의 CIDR 및 서브네팅(Subnetting)

과거에는 IP 주소를 A, B, C 클래스 단위로 덩어리째 낭비하듯 할당했습니다. C클래스 하나만 받아도 무조건 256개의 주소를 가져가야 해서, 10대만 쓰는 회사는 남은 246개의 주소를 버려야 했습니다.

- 현대 네트워크는 이를 타파하고 **CIDR(Classless Inter-Domain Routing)** 표준을 사용합니다.
- 가장 촘촘한 서브넷 마스크(예: `/29`, `/30`)를 활용해 딱 필요한 개수(예: 4개, 8개)만큼만 IP 주소 영토를 정밀하게 쪼개어 분배하는 서브네팅(Subnetting)을 수행함으로써, 네트워크 경계선에서 버려지는 무의미한 IPv4 자원을 제로에 가깝게 누수 차단합니다.

### 3. DHCP(Dynamic Host Configuration Protocol)를 통한 유동 IP 동적 할당

모든 기기에 IP를 고정(Static)으로 박아두면, 컴퓨터가 꺼져 있거나 사람이 퇴근해서 자리를 비운 순간에도 주소 자원이 독점되어 낭비됩니다.

- 이를 막기 위해 ISP(통신사)나 내부 라우터는 **DHCP 서버**를 운영합니다.
- IP 주소를 영구 소유하는 것이 아니라, 컴퓨터가 켜져서 네트워크에 접속하는 순간에만 제한된 시간(Lease Time) 동안 IP 주소를 **빌려주고(대여)**, 컴퓨터가 꺼지거나 연결이 끊어지면 즉시 IP를 수거하여 다른 사람에게 돌려막기식으로 재할당하는 공유 경제 스케줄링을 통해 주소 효율성을 극대화합니다.

요약하자면, 현대 네트워크는 IPv6로의 전면 전환 전까지 **사설 IP와 PAT(포트 다중화)를 통해 공인 IP 1개를 수천 배로 뻥튀기해 쓰고, CIDR 서브네팅으로 누수를 막으며, DHCP 유동 할당으로 자원을 실시간 회수**하는 삼중 방어선으로 IPv4 고갈 문제를 영리하게 버텨내고 있습니다.

## 🤔 질문 의도 및 답변 분석

### 질문 의도

면접관이 확인하고 싶은 골드 포인트는 두 가지입니다.

1. **현실적 제약의 이해**: 수많은 대기업 인프라와 라우터 장비, 레거시 시스템을 통째로 IPv6로 바꾸는 비용(호환성 문제)이 너무 막대하여 IPv4를 연명시켜야만 하는 현실을 인지하는가.
2. **주소 보존 기술(Address Conservation)의 실무적 체계**: 공인 IP 하나를 수백 명의 사설 IP 유저가 쪼개 쓰게 만드는 **NAT/PAT 아키텍처**와, 서브넷 마스크를 쪼개어 버려지는 주소를 최소화하는 **서브네팅**의 원리를 정확히 설명할 수 있는가.

### 답변 분석

- **실무 인프라의 핵심 메커니즘 도출**: 단순히 "공유기를 쓴다"가 아니라, 공인 IP 1개로 다중 접속을 여는 진짜 기술적 본질인 PAT (Port Address Translation)와 포트 매핑의 인과관계를 정확하게 논증했습니다.
- **과거와 현대 표준의 대조를 통한 설득력**: 과거의 비효율적인 클래스(Classful) 기반 주소 배분 방식의 한계를 짚고, 이를 타파한 현대의 **CIDR 서브네팅** 최적화 전략을 결합하여 전산학적 깊이를 완벽히 증명했습니다.

### 핵심 키워드

- 사설 IP 대역 지정 및 단일 공인 IP에 포트를 매핑해 재사용하는 NAT/PAT 기술
- 클래스 구분을 없애고 비트 단위로 주소를 촘촘히 쪼개는 CIDR 및 서브네팅(Subnetting)
- 컴퓨터가 네트워크를 쓸 때만 주소를 동적으로 대여하고 회수하는 DHCP 스케줄링
