## ❓질문

```text
두 호스트가 동시에 연결을 시도하면, 연결이 가능한가요? 가능하다면 어떻게 통신 연결을 수행하나요?
```

---
## 😎 답변 및 꼬리 질문

### 답변

네, 두 호스트가 서버와 클라이언트의 구별 없이 **동시에 서로를 향해 연결을 시도해도 완벽하게 연결이 가능합니다.** TCP 표준 스펙(RFC 793)은 이를 **'동시 오픈(Simultaneous Open)'** 메커니즘으로 정의하고 있으며, 이때는 일반적인 3-Way Handshake가 아닌 **총 4번의 패킷 교차가 일어나는 '4-Way 구조'로 통신 연결을 수행**하게 됩니다.

호스트 A와 호스트 B가 미리 서로의 IP와 포트 번호를 알고 있는 상태에서 동시에 연결을 찌를 때의 구체적인 통신 시퀀스는 다음과 같습니다.

1. **동시 SYN 전송 (`SYN_SENT` 상태 진입)**
    - 호스트 A는 호스트 B를 향해 `SYN(Seq=A)` 패킷을 보냅니다. 동시에 호스트 B도 호스트 A를 향해 `SYN(Seq=B)` 패킷을 보냅니다.
    - 이 시점에 두 호스트는 상대의 패킷을 아직 받지 못했으므로, 소켓 상태가 동시에 연결 요청 중임을 뜻하는 **`SYN_SENT`** 상태가 됩니다.
2. **네트워크 상에서 SYN 패킷의 교차 및 수신 (`SYN_RCVD` 상태 전환)**
    - 호스트 A가 보낸 `SYN`과 호스트 B가 보낸 `SYN` 패킷이 네트워크 통로 상에서 교차하여 각각 상대방에게 도착합니다.
    - 호스트 A는 `SYN_SENT` 상태에서 대기하다가 호스트 B의 `SYN`을 받게 되고, 소켓 상태가 **`SYN_RCVD`** 상태로 전이됩니다. 호스트 B 역시 호스트 A의 `SYN`을 수신하고 **`SYN_RCVD`** 상태로 바뀝니다.
3. **서로에 대한 확답(ACK) 전송**
    - 이제 양쪽 호스트는 상대방의 연결 요청(`SYN`)을 확인했으므로, 이에 대한 수락 확답인 **`ACK`** 패킷을 각자 다시 보냅니다.
    - 호스트 A는 호스트 B에게 `ACK(Ack=B+1)`를 보내고, 호스트 B 역시 호스트 A에게 `ACK(Ack=A+1)`를 보냅니다.
    - *주의할 점*: 이 패킷에는 새로운 `SYN` 플래그가 포함되지 않고 오직 **`ACK` 플래그만** 켜져 있습니다.
4. **최종 ACK 수신 (`ESTABLISHED` 연결 완료)**
    - 교차해서 날아간 최종 `ACK` 패킷을 양쪽이 서로 수신하는 순간, 호스트 A와 호스트 B의 소켓 상태는 비로소 통신 준비 완료를 뜻하는 `ESTABLISHED`로 동시 전환됩니다.

요약하자면, 일반적인 핸드셰이크는 `SYN` -> `SYN+ACK` -> `ACK`로 이어지는 3단계이지만, 동시 오픈 상황에서는 양방향 통로를 확립하기 위해 **`SYN` -> `SYN(교차)` -> `ACK` -> `ACK(교차)`의 4단계 패킷 교환**을 거쳐 단 하나의 정상적인 TCP 커넥션을 안전하게 수립하게 됩니다.

## 🤔 질문 의도 및 답변 분석

### 질문 의도

면접관이 이 질문을 통해 확인하고 싶은 핵심은 "TCP 소켓 상태 머신(State Machine)의 완벽한 이해"입니다.

양쪽이 동시에 `SYN`을 보내면 3-Way가 아닌 4-Way 형태로 패킷이 교차하는 특이한 핸드셰이크(Simultaneous Open)가 발생합니다.

이때 소켓 상태가 어떻게 꼬이지 않고 정상 연결(ESTABLISHED)에 도달하는지 패킷 타임라인을 꿰뚫고 있는지 검증하려는 것입니다.

### 답변 분석

- **고급 표준 스펙 명시**: RFC 793 표준에 정의된 공식 명칭인 **'동시 오픈(Simultaneous Open)'** 키워드를 정확히 터치하여 깊이감을 보여주었습니다.
- **소켓 상태의 정교한 추적**: `SYN_SENT` 상태에서 `LISTEN`을 거치지 않고 곧바로 상대의 `SYN`을 받아 `SYN_RCVD`로 전이되는 흔치 않은 소켓 상태 장치의 메커니즘을 타임라인별로 선명하게 증명했습니다.
- **패킷 수의 오해 교정**: 동시에 연결하면 소켓이 2개 열리거나 충돌이 날 것이라는 면접관의 고정관념 유도를 깨부수고, **4단계 패킷 교환을 통해 '단 하나의 커넥션'이 유기적으로 완성된다**는 진실을 명쾌하게 풀어냈습니다.

### 핵심 키워드

- TCP 동시 오픈 (Simultaneous Open, RFC 793)
- `SYN_SENT` 상태에서 서로의 `SYN`을 수신하여 `SYN_RCVD`로 전이되는 과정
- `SYN` -> `SYN` -> `ACK` -> `ACK` 구조의 4단계 패킷 동기화
